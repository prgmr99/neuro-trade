# NeuroTrade × Google AI Studio 결합 방향성 (1차 아이디어)

> Status: **Draft v1 — 1차 아이디어**
> Date: 2026-04-09
> Scope: NeuroTrade가 Google AI Studio(Gemini API)를 활용해 경쟁력 있는 시뮬레이션 게임으로 한 단계 성장하기 위한 전략 방향성

---

## 전제

단순히 "AI 붙인 게임"이 아니라, 지금 NeuroTrade의 **차별점(분석력 > 운, 시나리오 기반)을 증폭**시키는 방향으로 Gemini를 활용한다.

핵심 제약:
- 시뮬레이션 결정론(`mulberry32` 시드 기반) 유지
- 뉴스 effect 밸런스 규칙(Classic ±15%, Advanced raw ±30%) 유지
- API 키는 프론트 노출 금지 → Vercel/Supabase Edge Function 경유

---

## 4가지 전략 축

### 1. "무한 시나리오" — 콘텐츠 고갈 문제 해결 (최우선 리스크 해결)

**문제**: 현재 가장 큰 리텐션 리스크는 **정적 뉴스 풀**. Classic Arc Chain도 유저가 몇 번 돌면 외워버림.

**지향점**: Gemini를 **런타임 생성기**가 아니라 **오프라인 시나리오 팩토리**로 쓴다.

- `gemini-3-pro-preview` + **Structured Output(JSON schema)** 로 `Arc → News[]` 구조를 스키마 고정해서 배치 생성
- 생성 → 밸런스 자동 검증(effect 범위 체크) → 큐레이션 → `src/data/generated/`에 커밋
- **시드 결정론은 유지**: AI는 콘텐츠만 만들고, 시뮬레이션은 여전히 `mulberry32(seed)` 기반
- **Daily Challenge**: Google Search **Grounding**으로 오늘의 실제 거시 테마(금리, AI, 에너지)를 픽션화 → "오늘의 시장 스토리" 느낌

**왜 중요한가**: 결정론/밸런스/비용 3가지를 깨지 않으면서 콘텐츠만 10배로 확장.

---

### 2. "AI 트레이딩 코치" — 제품 정체성 강화

**문제**: NeuroTrade의 USP는 "운이 아니라 분석". 그런데 **플레이어는 왜 졌는지 모른다.** 게임 오버 화면에 수익률 숫자뿐.

**지향점**: 게임 종료 후 Gemini가 **개인화된 트레이드 리뷰**를 제공.

- 입력: 해당 세션의 `trades[]`, `news[]`, `pricePath`, 최종 포트폴리오
- 출력(구조화 JSON): `missedSignals[]`, `goodCalls[]`, `patternDiagnosis`, `nextTimeTry`
- 토스 스타일 카드 UI: "당신은 GREEN의 규제 뉴스를 놓쳤어요. 3일차에 손절했다면 +12%였을 텐데요."
- **반복 플레이 동기**를 "수익률 기록 경신" → "실력 성장 추적"으로 확장
- `achievementStore`와 결합하면 "코치가 본 내 약점 지표" → 장기 동기 루프

---

### 3. "AI 듀얼 상대" — 비동기 경쟁 확장

**문제**: 현재 Duel은 시드 공유 1:1, Multiplayer는 Supabase 실시간(인원 모객 이슈). 중간이 비어있음.

**지향점**: **페르소나 기반 AI 트레이더**를 상대로 언제든 듀얼.

- Gemini로 "공격형 추세추종가", "가치투자자", "컨트래리언" 등 성격 정의
- 각 페르소나는 같은 시드/뉴스에서 **의사결정 로그 생성**(매수/매도 타이밍 + 근거)
- 플레이어는 "박OO 애널리스트에게 이겨라" 같은 스토리 모드로 즐김
- 비동기라 매칭 대기 없음 → 한국 모바일 유저의 "5분 플레이" 패턴에 맞음
- 멀티플레이어 인프라(Supabase Realtime) 부하 없이 경쟁감 확보

---

### 4. "UGC 시나리오 에디터" — 장기 플랫폼화 포석

**문제**: 게임이 IP로 크려면 **유저가 만드는 콘텐츠**가 필요.

**지향점**: Gemini를 **유저의 시나리오 작성 도우미**로 노출.

- "2008 금융위기 스타일 5일 시나리오 만들어줘" → Gemini가 Arc/News 초안 생성 → 유저가 다듬어 공유
- 공유된 시나리오에 좋아요/플레이 수 → 랭킹
- AI가 밸런스 자동 검수(effect 범위, 종목 편향)

## 우선순위

| 순위 | 축 | 이유 |
|---|---|---|
| 1 | AI 트레이딩 코치 | 정체성 강화, 기존 세션 데이터만으로 즉시 구현 가능, 비용 저렴 |
| 2 | 무한 시나리오(배치 생성) | 리텐션 근본 해결, 오프라인이라 런타임 리스크 0 |
| 3 | AI 듀얼 상대 | 멀티플레이어 모객 문제 우회 |
| 4 | UGC 에디터 | 앞 3개 검증 후 착수 |

---

## 피해야 할 함정 (Anti-Patterns)

- **런타임 뉴스 생성**: 결정론 깨지고 밸런스 위반, 지연·비용·hallucination 3중고 → 반드시 오프라인 배치
- **"Gemini가 실시간 추천해주는 AI 트레이딩 봇"**: 투자 조언 경계선, 법적 리스크, 게임성 파괴
- **모든 텍스트를 AI 생성으로 전환**: 한국어 금융 카피의 결은 사람 손 필요. AI는 초안, 사람이 큐레이션
- **API 키 프론트 노출**: 반드시 Vercel Edge Function / Supabase Edge Function 경유

---

## 한 문장 요약

> Gemini는 "게임 엔진"이 아니라 **"시나리오 작가 + 코치 + 비동기 상대"** 로 쓸 때 NeuroTrade의 "분석력 기반 시뮬레이션"이라는 차별점이 증폭된다. 결정론과 밸런스는 그대로 두고 **콘텐츠·학습·경쟁 레이어만** AI로 확장하는 게 핵심.
